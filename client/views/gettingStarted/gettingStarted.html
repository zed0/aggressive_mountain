Getting Started

Getting the project

The source for this page is available on Github.
To edit this code you will need to download it from github using git. The command to do this is: 

git clone https://github.com/zed0/aggressive_mountain.git
This will download most of the code into a directory inside your current folder.
To check the project works open up the local client/index.html file in your browser, you should see another copy of this site. Because of chrome's security settings you will need to launch it with google-chrome --disable-web-security --allow-file-access-from-files.

This copy of the site is on my webspace, but if you want to see what it looks like online you can get free webspace from compsoc for this and other projects.

Modifying the project

The world is your oyster, open up your favourite editor and do whatever you want to the code. There are some ideas on the front page, but feel free to choose something else to do.
Kate and vim are installed on the machines, and Sublime or Atom are easy to install.

Structure

Try to maintain the structre of this project. If you are adding a modular reusable component that can be used in several places then put it in client/components/componentName, if you are adding new page views to the site then put them in client/views/viewName.
Don't be afraid to add subdirectories if you feel your code needs splitting up more, or if you're creating a sub-component of a larger feature.

Giving your changes to everyone else

Since this project is on Github we'll be using that to coordinate various people's changes. You can clone the repository, update that, and then make a pull request. If you don't know how to do this then there are some tutorials listed in the Resources section.
We'll try to respond to pull requests as fast as we can, but code review can be a slow process. Don't be offended if you get asked to make some changes before we merge your code, we're just trying to keep the code tidy for everyone.
